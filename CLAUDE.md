# Development Workflow

1. **Think high-level first** — before touching any code, map out the full scope of what needs to change. Identify all affected files, consider edge cases, and have a clear plan.

2. **Test-driven development** — write failing tests first (red), then implement just enough to make them pass (green). Never write implementation before tests.

3. **Deliver complete work** — ship coherent, complete changes. No half-baked incremental patches. Every commit should represent a fully working, fully tested unit of work.
